Introduction

React is a JavaScript library for building user interfaces, primarily for web applications. It is not a full framework but a library focused on creating reusable UI components. React allows developers to build interactive and dynamic web pages by breaking down the UI into modular pieces called components.

React is maintained by Meta and a community of developers. It renders UI efficiently using a virtual DOM, which updates only the parts of the page that change, improving performance.

React was first released in 2013, and its current stable version, as of 2025, is React 18, which introduced features like concurrent rendering and automatic batching.

Prerequisites

You don’t need prior React knowledge to start learning, but familiarity with JavaScript (ES6+), HTML, and CSS is essential. You should also have Node.js installed and be comfortable using a code editor like VS Code.

Getting Started

To start a React project, you can use Create React App or Vite to set up a development environment quickly.

npx create-react-app my-app cd my-app npm start

Example Explained

  • npx create-react-app my-app creates a new React project.
  • cd my-app navigates into the project folder.
  • npm start starts the development server.
  • The default App component will render a simple webpage at localhost:3000.

Components

Components are the building blocks of React applications. They are reusable pieces of UI, defined as JavaScript functions or classes.

function Welcome() { return <h1>Hello, World!</h1>; }

  1. A component is a function or class that returns JSX (a mix of JavaScript and HTML-like syntax).
  2. Components can be reused across your application.
  3. They must start with a capital letter to distinguish them from regular HTML elements.
  4. Components can be nested to build complex UIs.

Props and State

Props (properties) are read-only data passed to components to customize their behavior or appearance.

State is a local data storage within a component that can change over time, triggering re-renders.

function Counter() { const [count, setCount] = React.useState(0); return ( <div> <p>Count: {count}</p> <button onClick={() => setCount(count + 1)}>Increment</button> </div> ); }
React Basics

React applications are built by combining components into a tree structure. Each component can have its own logic and rendering.

Key concepts include:

  • JSX: A syntax extension for JavaScript to write HTML-like code.
  • Virtual DOM: A lightweight copy of the DOM for efficient updates.
  • Rendering: React updates the DOM when state or props change.

JSX Syntax

JSX allows you to write HTML-like code in JavaScript. It’s transpiled to JavaScript by tools like Babel.

Special considerations: <, >, ", ', and & are escaped in JSX. Use curly braces {} to embed JavaScript expressions.

Visit React JSX Docs for more details.

Event Handling

React handles events using camelCase attributes like onClick instead of HTML’s lowercase onclick.

Example: <button onClick={() => alert('Clicked!')}>Click me</button>

The event handler is a JavaScript function, and you pass it directly to the attribute.

Hooks

Hooks are functions that let you use state and other React features in functional components. Introduced in React 16.8, they replace class-based state management.

Common hooks: useState, useEffect, useContext.

Example: const [value, setValue] = React.useState('');

Metadata

The <head> of an HTML document in a React app contains metadata like the title and links to scripts or stylesheets. React itself doesn’t manage this directly, but tools like React Helmet can help.

Example: <title>My React App</title>

App Structure

React apps are structured as a tree of components. Common elements include:

<App>

  • Root component
    • <Header>
  • Header component
    • <Main>
  • Main content
    • <Footer>
  • Footer component

    • Styling React

    React components can be styled using CSS, inline styles, or libraries like Tailwind CSS or styled-components.

    Example of inline styles: <div style={ { backgroundColor: 'blue' } }>Styled Div</div>

    CSS files or modules are recommended for larger projects to keep styles organized.

    Debugging React

    React provides tools like React Developer Tools (a browser extension) to inspect component hierarchies and state. Errors often arise from incorrect state updates or prop mismatches.

    Common issues:

    • Key errors in lists (always provide a unique key prop).
    • State not updating (use immutable updates with setState).
